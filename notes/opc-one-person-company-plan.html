这篇文章是我用 OPC 一人公司专家团（主理人易牧 + 8 位领域成员）跑完「建盘期」01–07 阶段后的完整复盘。协作方式：每位成员只产出自己那段专业结论，由主理人汇编中转，全程不跳步、不代写。下面把每一阶段的结论和关键决策整理成一篇，方便以后照着执行、也方便随时回来复盘。

一、缘起与总览

我想从零规划一个一人公司，主线动作是公众号 + 视频号 + 网站运营（内容驱动），终局目标是带课变现（知识付费 / 课程）。这条路本身是成立的——内容矩阵做信任，课程做变现，是典型的「内容 → 信任 → 成交」结构。

但「想做」和「做对」之间差着一套方法论。于是我用了《一人企业方法论》第二版的 9 阶段协作流程，由专家团陪我从「资源盘点」一路走到「转化闭环」。建盘期（01–07）是线性、强依赖的：

建盘期 SOP 地图 01 资源盘点 → 02 利基定位 → 03 价值主张 → 04 商业模式 ↓ 06 MVP 设计 → 07 转化闭环 （阶段 05 已并入 04 的 Lean Canvas 内部设计，不单列）

一句话总览这套规划的核心结论：

我「生产内容」的能力远超「触达用户 / 锁定人群」的能力——资产与能力是真正的王牌，最大的不确定在「锁定哪类人」和「每周能投入多少时间」。所以整张地图的重心，是用利基定位把人群钉死，再用不依赖粉丝的获客路径，去对冲「触达极低」这个核心瓶颈。

二、阶段 01 资源盘点：摸清楚手里的牌

资源盘点就是先把「现在手里有什么」系统列清——而不是去想「将来要做什么」。它是整个流程的第一步，因为后面的利基、价值主张、带课设计全得立在这份家底上，否则是空中楼阁。方法论把创始人资源分成 8 大类，我用自己的长期画像做了预填，再补答了 10 个问题，最终定稿。

三、阶段 02 利基定位：锁定「做给谁」

利基（Niche）= 细分市场里「小而准」的切入点：某一类人 + 一个具体痛点 + 你刚好能填。它必须长在阶段 01 的资源上，不能凭空想。利基定位师用了「三环合一框架」来找最优缝：

  • 环 1 新杠杆 / 元杠杆：你有哪些「无需许可、复制成本趋零」的放大手段。我的王牌是元杠杆——已跑通的「每日自动化内容任务」+ 深度用 AI，等于用 AI 把每周 4–5 小时放大成自动运转的内容流水线。
  • 环 2 边界变动：行业什么边界在移动、大公司追不上。AI 让一人干一团队的活；技术社区崛起（搜索 + 垂直社区，不靠粉丝基数）。
  • 环 3 独有资源：取自阶段 01 的资产 / 能力 / 经验 / 人群。

三环重叠的最优缝 = 用 AI 自动化工作流 + 全栈能力，帮技术向副业探索者 / 独立开发者以极少时间跑通一人公司 / 副业，天然通向带课。

针对「触达极低」瓶颈（A 专门对冲）：① 利基精准度降获客难度 ② 资产复用做 SEO 搜索流量 ③ 跨平台自动化分发 ④ 借力技术垂直社区（内容即信任、不靠粉丝）⑤ 公开拆解自己的工作流做示范获客 ⑥ 元杠杆对冲人力。

四、阶段 03 价值主张：凭什么信你

价值主张用大白话说就是——「这类人凭什么信你、买你」。它是一座桥，左头接阶段 02 的利基靶心，右头接你具体交付什么、帮他们拿到什么结果、卸掉什么痛。本阶段用 Jobs / Pains / Gains 框架拆解目标人群（技术向副业探索者/独立开发者）：

  • Jobs（顾客任务）：用极少时间（4–5h/周）跑通自动持续产出；把零散 AI 工具串成可复用流水线；持续累积可变现资产；在技术圈被认可是「真把 AI 用起来的人」。
  • Pains（痛点）：工具零散串不起来、缺自动跑的骨架；搭两次半途而废；不知从哪步开始（选择瘫痪）；怕踩坑/被割韭菜；一人干全栈+内容+运营精力被撕碎；看别人搞副业自己还在零散用 → 焦虑自我怀疑；想学没人带。
  • Gains（收益）：开箱即用的可跟做工作流骨架；方法论可复制迁移；用最少时间放大成稳定输出；资产持续累积；被真实案例带练、不焦虑；有人带不孤独；获得「会用 AI 杠杆的技术个体」身份认同。

供给侧（Products & Services）：① 亲身验证方法论内容（公开拆解自己跑通的流水线）② 可复用工作流模板/脚手架 ③ 陪练形态（答疑/跟做反馈/轻社区）④ 公开拆解示范获客 ⑤（终局）体系化课程。

我拍板：就按推荐的来

价值主张陈述（定稿）：为想用 AI 放大副业产出、却没时间搭系统的技术向副业探索者/独立开发者，在「从零散用 AI → 自动运转工作流」的落地断层与试错焦虑上，通过主理人亲身跑通并持续验证的 AI 杠杆化工作流方法论、可复用模板与陪练，带来一套用最少时间就能自动持续产出的工作流骨架，以及「真用起来、不踩坑、有人带」的掌控感与长期安心。

六、阶段 06 MVP 设计：先验证两个命门

阶段 05 已并入 04 的 Lean Canvas 内部，所以 04 之后直接是 06。MVP（最小可行性产品）设计的核心是：用最小动作，先验证最致命的两个假设 R1（有没有人）和 R2（愿不愿付），而不是一上来就做完美产品。

R1 触达验证（第一个最小实验）

  • 最小动作：发 10–20 篇「公开拆解你自己跑通的 AI 自动化内容流水线」图文 + 搭 1–2 个 SEO 落地页（appin.site + GitHub）
  • 首发：掘金 + 知乎（精准+SEO）→ 小红书+GitHub 承接 → 视频号后置（首个 90 天不做视频，图文最省）
  • 时间盒：30 / 60 / 90 天三节点；阈值占位：90 天月增精准私域 X=30–50/月（锚40）、公域→私域率 Y=1.5%–3%（锚2%）

R2 付费意愿验证（精选模板包探针）

  • 最小动作：上架低价「AI 自动化内容流水线脚手架」模板包（prompt集 + n8n/Make JSON + 分发清单 + SEO 模板，复用元杠杆资产）+ 训练营早鸟/预售（定金锁位）
  • 定价占位：模板包 ¥19.9–29（可 ¥9.9 引流+¥49 组合）；早鸟定金 ¥49–99
  • 阈值占位：模板包付费率 Z=3%–8%（锚5%）、早鸟 W=10–20 人（锚15）
  • 关键纪律：模板包（弱意愿）和早鸟定金（强意愿）都达标 R2 才算成立——只卖动模板包 ≠ 愿为「工作流+陪练」付费

验证优先级路线图 + 北极星指标

阶段验证什么最小动作 + 停止/继续标准
P0（1–12周）R1+R2有没有人、愿不愿付；未达阈值→回炉渠道/价值主张
P1（P0后）R3+R4接不接得住（内容→加微率/私域→首购率）、撑不撑得住（周耗时≤4–5h）
P2（P1后）R5首期最小规模训练营（10–20人）交付成本/完课率/满意度

北极星指标= 精准私域月净增数（冷启动期「先蓄水再收网」；付费是二阶结果，依赖私域基数）。护栏指标 = 付费探针转化率 Z（样本小噪音大，不作主北极星）。

七、阶段 07 转化闭环：从被看见到成交

阶段 07 是建盘期收官——把阶段 06 验证过的分散动作，连成一条触达 → 承接 → 成交的转化路径。先证真伪（06），再连路径（07）。

① 触达 Reach（基于 R1 实验）

在冷启动、无关系网、不依赖粉丝基数的前提下，用「公开拆解示范」拿到稳定精准触达。公开拆解文本身就是钩子内容——它既是获客动作，也是版本 2 信任主打的载体（真在跑、不割韭菜）。

渠道角色
掘金、知乎第一波首发（精准 + SEO 友好）
小红书承接 / 截流，导流落地页
GitHub资产 + 代码承接（README + 真实可运行脚手架）
V2EX / 少数派 / 独立开发者社区精准补充（画像极准）
视频号 / 抖音后置（高成本，R1 跑通后再上）
自有渠道（公众号18/视频号100/appin.site）只接住 + 复触达，不拉新

② 承接 Capture（公域读者转私域）

把读完拆解文的人，变成「你随时能触达」的私域联系人。这一步是闭环的咽喉——公域流量是平台的，私域才是你自己的。承接载体：落地页加微入口 / 社群 / IMA 知识库 / 公众号菜单；基调统一走版本 2 信任（真在跑 / 不割韭菜 / 有人带）；CTA 框架：每篇钩子文末尾 + SEO 页底部 + GitHub README 内置「领完整脚手架 / 进群一起跑」入口。指标：X（月新增精准私域·北极星）、加微/进群数、Y（公域→私域率）。

③ 成交 Purchase（私域 → 首购 → 升级）

4 级跃迁：低价模板包（弱信号 Z）→ 训练营早鸟锁位（强信号 W）→ 正课（主收入）→ 体系课/陪跑（升级）。

私域池 ├─ 节点1 · 弱意愿：低价「精选模板包」 ¥19.9–29（R2 弱信号） ├─ 节点2 · 强意愿：训练营早鸟/预售（定金锁位）¥49–99（终局正课 ¥299–999/期） ├─ 节点3 · 主收入：训练营正课（首期最小规模 10–20 人，P2 验 R5） └─ 节点4 · 升级：体系课（常青）+ 陪跑（限量）

成交主张 = 信任型（有人带 / 不踩坑 / 低风险阶梯）。R2 判定纪律：模板包（弱）与早鸟（强）都达标，R2 才成立。

转化漏斗 + 飞轮（自循环）

公域曝光 → 阅读 → 加微/进群(私域·北极星X) → 模板包付费(弱Z) → 早鸟锁位(强W) → 训练营正课 → 体系课/陪跑 飞轮： 学员成交 → 案例/见证反哺内容 → 强化「公开拆解示范」获客 → 更多精准触达→更多加微(X)→更多成交→更多学员/见证 … 红线：信任信号（学员见证、真实跑通）须真人/真实产生，AI 不得编造未发生的成果。

八、建盘完成 · 进入执行期

阶段 07 落盘后，建盘期（01–07）全部完成。现在进入执行阶段——按上面这条转化路径实际去做。以下两种情况，再回来找专家团：

  • 运营卡住、找不到问题在哪 → 触发经营复盘（复明），用轻量指标找瓶颈、定下周期重点
  • 有东西开始重复出现、想系统化 → 触发资产沉淀（沉墨），把可复用成果规划成复利资产

整张地图的重心始终没变：用「资产 + 能力」的高杠杆去对冲「触达极低」的核心瓶颈。所以执行期第一步不是做产品，而是发 10–20 篇公开拆解文 + 1–2 个 SEO 落地页，先验证 R1（触达）和 R2（付费意愿）这两个 P0 命门。30 天基线出来后，再回推校准 X/Y/Z/W 的阈值。

一句话收尾：内容生产我这边长板很明显，短板在触达和信任。所以整盘生意的设计逻辑是——用「真在跑的系统 + 有人带」建立信任，用「不靠粉丝、靠利基精准度 + 技术社区 + 资产复用」突破触达，最后用「模板包→早鸟→训练营→体系课」把信任转化成可持续的带课收入。
