PXID 是 B2B 电助力车 ODM/OEM 出口站，SEO 主战场是英文 Google。这份路线图把"怎么开展"拆成可落地的九步——研究底座来自团队关键词研究员关宇霖的竞品实测。

核心结论（TL;DR）

  • 北极星：不拼头部词，赢长尾才是正道——"OEM/ODM + 品类 + 地理/合规"长尾 KD 仅 25–40，意图强、易转化。
  • 预算分配：7 成压「商业调查型 + 交易型」长尾，3 成用「信息型指南」养权重。
  • 内容架构：1 个 Pillar 枢纽页串 8 篇 Cluster，并把现有 News 升级为主题集群，避免内容孤岛。
  • 差异化核武器："中国 vs 印度制造 / 反倾销关税"近零竞争、极高价值，做成支柱级内容。
  • 超车缝隙：竞品普遍"有 FAQ 无指南、有产品无教育"，正是 PXID 可抢的位置。

一、关键词与意图布局

  • 信息型：what is OEM/ODM、certifications explained → 主攻（顶部漏斗养信任）

  • 交易型：e-bike OEM manufacturer China、custom e-bike factory、cargo e-bike OEM → 核心主攻（承接产品页+询盘，意图最强）
  • 导航型：PXID、pxid.com → 守住品牌词即可

二、内容引擎：Pillar–Cluster 主题集群

把"写一堆散文章"升级为"一个枢纽页串起 N 篇集群"，既养权重又防内容蚕食。

蚕食防护：C5/C6 按品类、C1/C2/C8 按决策阶段、C4/C7 按合规/地理切面切分，主词唯一、互不重叠；内链用描述性锚文本回指正确页面。

2.2 把现有"新闻"升级为"主题集群"

PXID 已有高质量 News（EUROBIKE 2026、印度策略、营收 +484%），质量已超多数竞品 Blog。WordPress 迁移后，将 News 重分类打上聚类标签，描述为 Pillar–Cluster 并做描述性互链，避免内容孤岛。

三、站内技术 SEO（结合 WordPress 迁移 + COS 部署）

这是"开展"的硬落地部分，直接对应你正在做的迁移：

  • Core Web Vitals：迁移后重点看 LCP / CLS / INP。COS 静态托管对速度友好，但 WP 动态页需配缓存 + CDN（腾讯云 CDN 或 COS+CDN）。图片统一 WebP + 懒加载。
  • URL 结构与 301 重定向：迁移时务必做旧 slug → 新 slug 的 301 映射表，避免丢历史权重。产品页用 /products/[category]/[model]/ 层级。
  • 多语言 / 区域 (hreflang)：出口 30+ 国，建议 hreflang 标注 en / zh / 小语种（WPML 或子目录 /en/ /de/）。hreflang 做对可避免重复内容惩罚。
  • 结构化数据 (Schema.org)：标注 Organization / Manufacturer / Product / Offer / Breadcrumb，争取富媒体结果（星级、面包屑）。
  • XML Sitemap + robots.txt：迁移后重新提交 sitemap 到 Google Search Console (GSC)。
  • 阿里国际站内容同步的 canonical 处理：原阿里站产品图文同步到官网时，官网文案须做差异化（官网侧重 OEM/ODM 能力 + 案例，阿里侧重规格 + 价格），并对可能重复的部分设 canonical 指向官网，避免被判复制阿里内容。
  • HTTPS 全站：COS + 证书，混合内容清零。
  • 整站紧凑布局：你已把模块间距缩 50%，利于阅读密度；但需确保移动端（B2B 买家常手机查厂）响应式无误、按钮可点。

四、内链与架构（基于聚类）

  • Pillar 回链所有 C1–C8；Cluster 间按语义互链（C1 OEM vs ODM ↔ C2 选型清单 ↔ C7 中印对比）。
  • 锚文本用描述性短语（如 "custom e-bike development process"），禁用"点击这里"。
  • 主导航 / 首页增加 "OEM/ODM Guide" 入口，承接顶部漏斗流量。

五、站外 / 外链（B2B 出口）

  • 高价值外链来源：行业目录、电助力车媒体（Electrek / Bike Europe / EV25）、展会官网（EUROBIKE 参展商页）、LinkedIn 公司页、客户案例互链（经授权）。
  • 数字公关：把 C7「中国 vs 印度制造 / 反倾销关税」做成独家角度，易被行业媒体引用 → 自然外链（这是 PXID 的差异化核武器）。
  • 平台侧重：阿里国际站 / 中国制造网作次级曝光，但外链权重低，官网为主。
  • 严禁：PBN、垃圾目录、买链。

八、监测与迭代

  • 工具：Google Search Console（必备免费）+ Google Analytics 4；付费用 Ahrefs/Semrush 拉真实搜索量校准估算值。
  • 指标：自然流量、长尾词排名、询盘转化（表单/WhatsApp/邮件）、Core Web Vitals。
  • 节奏：每月复盘，季度调整聚类。

九、本周就能做的 5 件事

  1. 建 GSC 并验证 pxid.com，准备提交 sitemap。
  2. WordPress 迁移时做好 301 重定向映射表（旧 slug → 新 slug）。
  3. 把现有 News 文章打上聚类标签，准备重组为主题集群。
  4. 列出现有量化素材（营收 +484%、上市 -30%、售后 -40%、200+ 专利、印度工厂）做案例库。
  5. 用 Ahrefs 拉一次真实搜索量，校准研究 Brief 的估算值。
