WebNovel Writer：让 Claude Code 写出不崩人设的 200 万字长篇

一个跑在 Claude Code 上的长篇网文创作插件。它要解决的只有一件事——让 AI 写到几百章，依然记得住设定、接得住伏笔、守得住大纲。

一、它解决什么问题

长篇创作最难的不是写出第一章，而是写到第 80 章、第 200 章以后仍然保持：角色动机不漂移、战力/时间线/世界观不互相打架、伏笔有登记有回收、爽点与感情线保持节奏。普通 AI 写作写到十几章就开始「遗忘」和「幻觉」——把前面设定忘光，或凭空编造与已写内容冲突的情节。

WebNovel Writer 把上面这些「必须记住、不能写崩」的约束，变成 Claude Code 会自动执行的步骤：动笔前先查资料，写完后把新发生的事实记下来、做一致性审查，再同步进检索索引、章节摘要、长期记忆和可视化面板。它不只是一只会写的模型，而是边写边攒的一致性系统。

一句话定位：面向长篇连载的一致性系统，不是写完就忘的一次性生成器。官方称其支持 200 万字量级 的连载创作。

二、核心能力：8 个 Skill 命令

整条创作流程被拆成 8 个 Claude Code Skill 命令，从初始化到审查、查询、沉淀记忆，串成一条闭环：

1/webnovel-init · 深度初始化

分阶段问答，帮你把书的骨架、设定集、总纲和初始状态搭起来。

2/webnovel-plan · 卷纲规划

基于总纲拆卷、拆章、补时间线，并写回新增设定。

3/webnovel-write · 章节创作

一条龙写完一章：备上下文、起草、审查、润色、记录事实、自动备份。

4/webnovel-review · 质量审查

从爽点、一致性、节奏、OOC、连贯性、追读力等维度审查章节。

5/webnovel-query · 状态查询

查询角色、伏笔、节奏、实体关系和运行时信息。

6/webnovel-learn · 项目学习

把这本书里好用的写法记下来，存进项目长期记忆。

7/webnovel-dashboard · 可视化面板

只读浏览项目状态、实体图谱、章节内容和追读力数据。

8/webnovel-doctor · 项目体检

阶段感知检查目录、文件、数据库、RAG、依赖和 Dashboard 产物。

三、Story System：一致性引擎

v6.0.0 起上线的 Story System 是整套系统的核心，它把「怎么写」和「写了什么」分开：文笔和节奏可以放开发挥，但发生过的事实必须登记、过审、存档，不能含糊。

  • .story-system/：唯一的事实源头，动笔前的「合同」和写完后的「提交」都存在这里。
  • accepted 的 CHAPTER_COMMIT：一章写完，新事实从这里入账。
  • .webnovel/state.json、index.db、summaries/、memory_scratchpad.json：都是从主链派生出来的只读视图，供查询和展示用。
  • projection_log.jsonl：投影执行日志，用来定位 state / index / summary / memory / vector 哪一路没同步。

「边写边攒」——每章写完后事实会沉淀到可检索的状态系统，后续章节自动调用，这才是它对抗遗忘和幻觉的真正武器。

四、快速开始

通过 Claude Code Marketplace 安装插件，再装 Python 依赖并初始化一本书：

1. 安装插件

claude plugin marketplace add lingfengQAQ/webnovel-writer --scope user

claude plugin install webnovel-writer@webnovel-writer-marketplace --scope user

2. 安装 Python 依赖

pip install -r https://raw.githubusercontent.com/lingfengQAQ/webnovel-writer/HEAD/requirements.txt

3. 初始化一本书

在 Claude Code 中输入 /webnovel-init，会创建书项目目录（含 .story-system / 正文 / 大纲 / 设定集 / 审查报告）。

4. 配置 RAG（可选但推荐）

复制 .env.example 为 .env，填写 Embedding（Qwen3-Embedding-8B）与 Rerank（jina-reranker-v3）Key。没填也能用——系统自动退回 BM25 关键词检索，只是语义召回弱一些。

五、写章工作流（9 步流水线）

/webnovel-write 不是把活儿丢给模型生成一次就完事，而是一条带关卡的完整流水线：

1预检

项目根、占位符和 Story System 健康状态。

2刷新运行时合同

本章 runtime contract。

3context-agent 生成任务书

明确本章要写什么、约束是什么。

4起草正文

根据任务书落地章节内容。

5reviewer 多维审查

blocking issue 不通过则阻断。

6润色 · 排版 · Anti-AI 终检

去 AI 味，保证可读性。

7data-agent 提取事实

把新发生的事实结构化。

8生成 CHAPTER_COMMIT

驱动 state、index、summary、memory、vector 投影。

9章节级备份

落盘存档，可断点续写。

六、我的判断

适合：长篇连载网文作者、已经用 Claude Code 写作的人、受困于 AI「写到十几章就忘了前面设定」的人。内置 37 个中文网文题材模板（修仙、系统流、都市异能、古言宫斗、规则怪谈、抗战谍战……），也支持复合题材。

注意：① 强依赖 Claude Code 环境；② RAG 语义检索需 API Key，不填则降级 BM25；③ 概念较多，初次上手有学习曲线，建议先吃透 init → plan → write 三条主线。

定位：它不是「输入一句话吐一章」的一次性生成器，而是一套面向长篇连载的一致性系统——把设定、伏笔、节奏变成 AI 会自动遵守的约束。如果你写的是百万字级别、人设和世界观不能崩的长篇，它比通用写作助手靠谱得多。
