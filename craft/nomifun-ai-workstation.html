一、这是什么？

NomiFun 是一款完全开源的本地优先 AI 超级工作站。你下载一个桌面应用，就能拥有：

  • 🐾 桌面 AI 伙伴（自定义形象，桌宠既视感）
  • 🤖 无人值守 Agent（挂机自动跑任务，断了自动重试）
  • 📚 企业级知识库（读写分离，网页抓取，数据自由导出）
  • 🌐 11 个 IM 渠道（微信/钉钉/飞书/Slack 都能连）
  • 🔧 原生工具引擎（Rust 自研，省 Token，直接控制电脑）

一句话：不是玩具桌宠，是真正能干活的开源 AI 工作站。

二、核心功能（9大模块）

1. 🔒 安全基础：数据全在你硬盘上

所有数据存本地 SQLite，唯一出站请求是你配置的 LLM API 调用。无云账号、无遥测、无订阅。代码完全开源可审计。

2. 🐾 AI 伙伴系统：一个大脑，多张面孔

能力说明
内置同伴Mochi（年糕兔）、Ink（琥珀眼猫）、Bolt（悬浮机器人），全部手绘代码生成
自定义形象上传任意图片作为伙伴外观（亲人照片、宠物均可）
共享记忆所有伙伴共享记忆中枢
独立技能每个伙伴隔离的技能库
技能进化从工具调用序列自动挖掘可复用技能，生成 SKILL.md，用户确认后启用
技能赠送一个伙伴的技能可赠送给另一个伙伴

3. 🤖 无人值守自动化（AutoWork + IDMM）

这是核心差异点——让 NomiFun 在你离开电脑后依然工作：

  • 需求看板：待处理 → 进行中 → 完成/失败/待审查
  • AutoWork：自动认领任务 → 执行 → 循环轮转
  • IDMM 监督：Rule 层自动重试/退避；Sidecar model 层处理决策卡点
  • Lease sweeper：每 60 秒清扫孤儿任务
  • Boot resume：重启后恢复工作
  • 完成通知：飞书/Slack/HTTP webhook

4. 📚 统一知识库管理

能力说明
Markdown 目录用户策划的知识目录，挂载到会话工作空间
Live URL 快照网页转知识条目，SSRF 防护 + 无头浏览器处理 JS 页面
飞书源Feishu 作为知识源已实现
安全写回Disabled / Staged / Direct 三级，IM 写入始终进 _inbox 审查暂存区
一次配置处处用知识库、技能、Agent、MCP、模型统一配置后任意会话复用

5. 🖥️ 原生工具引擎（Rust 自研）

这是架构亮点——不是包装 OpenAI 的工具调用，也不是套 Playwright：

工具实现方式动作数
Browser自研进程内 Rust CDP 引擎，直接驱动 Chromium~32
Computer屏幕捕获 + 输入合成，macOS AX / Windows UIA / Linux AT-SPI~21
内置工具Read、Write、Bash、Grep、Glob、Lsp、Terminal9 项

无 Playwright、无 Node 子进程，极省资源。

6. 🌐 开放能力生态（MCP + REST）

  • 151 项能力通过三大门面暴露
  • /mcp — 全量 Remote 能力面（MCP Streamable-HTTP）
  • /mcp-agent — 精选 do-work 子集
  • /v1 — REST + OpenAPI 3.1 + SSE 流式响应
  • Claude Code / Codex / Cursor 等直接对接

7. 💬 11 个 IM 渠道

Telegram · 飞书 · 钉钉 · 微信 · Discord · Slack · Matrix · Mattermost · Twitch · Nostr · QQ Bot

企业微信进行中。一个伙伴绑定一个机器人，聊天即操控电脑。

8. 📱 WebUI 远程控制

局域网/VPN 下手机或平板扫码连接（一次 Token，5 分钟 TTL），获得与桌面完全一致的体验。

9. 🧩 模型支持

4 原生后端：Anthropic、OpenAI 兼容、Amazon Bedrock、Google Vertex。OpenAI 兼容可覆盖 26+ 提供商（DeepSeek、Gemini、Qwen、Kimi、Mistral、Ollama、vLLM 等）。17 ACP 直连代理：Claude Code、Codex CLI、Gemini CLI、Qwen、Goose、Cursor、Copilot、CodeBuddy 等。

四、怎么用？（小白版）

方式一：下载安装包（推荐）

下载地址：github.com/nomifun/nomifun-tauri/releases/

方式二：从源码构建

# 前置条件：安装 Rust（stable）、Bun（≥ 1.3.13）
git clone https://github.com/nomifun/nomifun-tauri.git
cd nomifun-tauri
bun install

# 开发模式（热重载）
bun run dev

# 构建安装包
bun run build

方式三：Docker 自托管

docker compose up -d --build
# 打开 http://<server-ip>:8787

配置步骤

  • 安装后打开 NomiFun，首次启动进入伙伴选择页
  • 配置模型：在模型管理页添加 API Key（Anthropic / OpenAI / DeepSeek 等任选）
  • 启航：选择伙伴角色或自定义形象，进入会话即可使用

十、总结

NomiFun 的核心差异点：

  1. 不是包装 AI API 的套壳工具——自研 Rust 浏览器引擎、原生计算机控制、IM 渠道网关，技术深度远超同类
  2. 不是封闭的 SaaS——Apache-2.0 完全开源，SQLite 本地存储，数据主权在你手里
  3. 不是一次性工具——AutoWork 无人值守 + IDMM 监督 + 需求看板，挂了也能跑

适合追求数据安全、需要长期复用的重度 AI 用户，尤其是开发者、企业团队、科研人员。

一句话：桌宠只是外壳，真正的价值是这个开源 AI 工作站的底层架构和无人值守自动化能力。
