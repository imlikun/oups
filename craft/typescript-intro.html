TypeScript 入门：给 JavaScript 项目加上类型安全

从 JavaScript 切换到 TypeScript 最大的障碍不是语法，而是思维方式的转变。分享一下我是如何克服初期挫折感的。

李坤 2026-05-10 约 419 字

为什么要学 TypeScript

写了半年纯 JavaScript 之后，最痛苦的是： - 函数参数传错类型，运行时才报错 - 重构时不知道哪些地方用了某个接口 - 团队协作时，不知道同事写的函数的参数格式

TypeScript 能在编译阶段就发现这些问题。

渐进式迁移策略

不需要一次性把所有文件改成 .ts/.tsx：

  1. 先给新文件用 TypeScript
  2. 旧文件慢慢改，改一个是一个
  3. 用 // @ts-ignore 临时跳过难改的部分

最有用的几个特性

  • interface：定义数据结构，重构时自动提示
  • 泛型：写可复用函数时非常有用
  • 类型推导：其实不用每个变量都写类型，TS 能自动推导

踩过的坑

tsconfig.json 的配置项非常多，一开始建议用 strict: true，不要为了省事关掉类型检查。
