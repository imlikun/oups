一键装上互联网能力

为什么需要 Agent Reach

AI Agent 已经能帮你写代码、改文档、管项目——但你让它去网上找点东西，它就抓瞎了：

  • "帮我看看这个 YouTube 教程讲了什么" → 看不了，拿不到字幕
  • "帮我搜一下推特上大家怎么评价这个产品" → 搜不了，Twitter API 要付费
  • "去 Reddit 上看看有没有人遇到过同样的 bug" → 403 被封，服务器 IP 被拒
  • "帮我看看小红书上这个品的口碑" → 打不开，必须登录才能看
  • "B站上有个技术视频，帮我总结一下" → 拿不到，通用下载工具被 B站风控拦截
  • "帮我看看这个网页写了啥" → 抓回来一堆 HTML 标签，根本没法读

这些不难实现，但每个平台都有自己的门槛——要付费的 API、要绕过的封锁、要登录的账号、要清洗的数据。你要一个一个去踩坑、装工具、调配置，光是让 Agent 能读个推特就得折腾半天。

Agent Reach 把这件事变成了一句话——复制给 Agent，几分钟后它就能读推特、搜 Reddit、看 YouTube、刷小红书了。

它到底是什么

Agent Reach 不是一个工具，而是一个能力层（capability layer）。它负责选型、安装、体检、路由，不负责底层读取本身。

简单说：它帮你选好"当下最稳的接入方式"，替你装好、体检好。接入方式会换代，你不用操心。

关于"零配置"

6 个渠道开箱即用（网页/YouTube/RSS/GitHub公开仓库/B站搜索/V2EX/雪球），无需任何配置。Twitter、Reddit、小红书等需要登录态的平台，告诉 Agent"帮我配 XXX"即可一步步引导完成。

真实案例：B站风控切换

2026 年 6 月，yt-dlp 被 B站风控 412 封死。Agent Reach 自动切换到 bili-cli（无需登录可搜可读），用户零操作。这就是"能力层"的价值——接入方式会换代，你不用操心。

一句话安装

复制这句话给你的 AI Agent（Claude Code、OpenClaw、Cursor 等）：

帮我安装 Agent Reach：https://raw.githubusercontent.com/Panniantong/agent-reach/main/docs/install.md

就这一步。Agent 会自己完成剩下的所有事情：

  1. 安装 CLI 工具

    pip install 装好 agent-reach 命令行（自带 yt-dlp、feedparser）

  2. 安装系统基建

    自动检测并安装 Node.js、gh CLI、mcporter

  3. 配置搜索引擎

    通过 MCP 接入 Exa（免费，无需 API Key）

  4. 检测环境

    判断是本地电脑还是服务器，给出对应配置建议

  5. 注册 SKILL.md

    在 Agent 的 skills 目录安装使用指南，以后 Agent 遇到"全网调研""搜推特""看视频"这类需求，会自动知道该调哪个上游工具

  6. 问你要不要更多

    默认只激活 6 个零配置渠道；小红书、Twitter、Reddit 等需要登录态的，Agent 会列菜单问你要哪些，点名才装

安装完之后，agent-reach doctor 一条命令告诉你每个渠道的状态、当前走哪条路。

三种安装模式

方式命令适合场景
一键全自动（默认）agent-reach install --env=auto个人电脑、开发环境
安全模式agent-reach install --env=auto --safe生产服务器、多人共用机器
仅预览agent-reach install --env=auto --dry-run先看看会做什么

OpenClaw 用户注意：安装前需先开启 exec 权限——openclaw config set tools.profile "coding"，然后重启 Gateway。Claude Code / Cursor / Windsurf 不受此限制。

设计理念：能力层而非工具

Agent Reach 比任何具体实现高一层——负责选型、安装、体检、路由，不负责底层读取本身。读取由 Agent 直接调用上游工具完成，没有包装层。

每个平台 = 首选 + 备选的有序后端列表

换接入方式 = 调整列表顺序，不是重写代码。agent-reach doctor 会告诉你每个平台当前在用哪个后端。

场景首选备选为什么这么选
读网页Jina Reader—免费，不需要 API Key
读推特twitter-cliOpenCLI实测搜索稳定；OpenCLI 走浏览器登录态兜底
RedditOpenCLI（桌面）rdt-cli匿名接口已被封，只剩登录态路线
YouTubeyt-dlp—154K Star，YouTube 最佳选择
B站bili-cliOpenCLIyt-dlp 被风控封死，bili-cli 无登录可搜可读
搜全网Exa via mcporter—AI 语义搜索，MCP 接入免 Key
小红书OpenCLI（桌面）xiaohongshu-mcp浏览器登录态零摩擦

这些都是"当前选型"，基于真机实测定期复核。某条路失效了就换下一条——agent-reach doctor 永远告诉你现在走的是哪条。

安全设计

措施说明
🔒 凭据本地存储Cookie、Token 只存在本机 ~/.agent-reach/config.yaml，文件权限 600，不上传不外传
🛡️ 安全模式--safe 不会自动修改系统，只列出需要什么，由你决定装不装
👀 完全开源代码透明，随时可审查。所有依赖工具也是开源项目
🔍 Dry Run--dry-run 预览所有操作，不做任何改动
🧩 可插拔架构不信任某个组件？换掉对应的 channel 文件即可，不影响其他

⚠️ 封号风险提醒：使用 Cookie 登录的平台（Twitter、小红书等），通过脚本/API 调用存在被平台检测并封号的风险。请务必使用专用小号，不要用你的主账号。Cookie 等同于完整登录权限，用小号可以在凭据泄露时限制影响范围。

适合谁用

  • 已在用 Claude Code / OpenClaw / Cursor 等 AI Agent 的开发者，需要让 Agent 能访问互联网内容
  • 需要跨平台数据采集的研究人员（Twitter 舆情、Reddit 讨论、小红书口碑）
  • 需要自动化内容监控的自媒体创作者（YouTube 字幕、B站视频摘要、RSS 订阅）
  • 对数据隐私敏感、不想把 Cookie 交给第三方云服务的用户
  • 不想为每个平台单独付费 API、不想踩反爬坑的效率工具爱好者

能力边界

Agent Reach 解决的是"读"的问题——读取内容、搜索信息、提取数据。如果你需要"动手"操作网页（登录后的表单提交、多账号隔离、自动化流程中的验证和风控提示），可以配合 BrowserAct 这类浏览器自动化工具使用。
