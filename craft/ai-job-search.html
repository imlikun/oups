AI Job Search：把 Claude Code 变成你的本地求职流水线

一个跑在 Claude Code 里的开源求职工作流。它把「刷岗位、算匹配度、改简历、写求职信、过 ATS、准备面试」这些重复劳动，编排成一条可被 AI 执行、验证、迭代的标准化流水线——数据留本地，投递你自己确认。

一、它解决什么问题

求职本质上是一件高重复、高主观、多步骤的事：每天刷岗位、把 JD 和自己的履历逐条比对匹配度、为不同岗位改不同的简历、写语气不同的求职信、再检查 PDF 排版和 ATS 文本层、最后准备面试。人肉做这些，既累又容易漏。

AI Job Search 把上面这套流程，变成一条跑在 Claude Code 里的可编程流水线。它的本质是一个基于 Claude Code 的 Agentic 工作流编排系统——把求职拆成一系列可被 AI 执行、验证和迭代的标准化流程，而不是一个替你海投的机器人。

一句话定位：把求职从「人肉重复劳动」变成「本地可审计、可迭代的 AI 流水线」。官方框架与地域无关，内置门户偏丹麦市场，但可通过 /add-portal 扩展到任意地区（含中国区职位平台）。

二、核心命令体系

整条求职流程被拆成一组 Claude Code 命令，从档案初始化到反馈校准，串成闭环：

1/setup · 个人档案初始化

三种路径：导入文档文件夹、导入单份 CV、或面试模式，先把你的背景结构化。

2/scrape · 多门户批量搜岗

跨职位门户批量搜索，去重后按匹配度排序呈现，一眼看清今天新出的机会。

3/rank · 批量评分排序

对所有抓取到的岗位批量评分，生成排名短名单，支持并行 Agent 评估提升效率。

4/apply · 核心投递工作流

匹配评估 → 简历定制 → 求职信 → 审稿人评审 → PDF 编译 → ATS 验证，六步一气呵成。

5/expand · 丰富技能档案

从 GitHub、作品集、Google Scholar 等公开来源补全你的技能与项目经历。

6/upskill · 技能差距分析

对照目标岗位生成优先级热力图与带时间估算的学习计划，知道该补什么。

7/outcome · 结果归档与校准

记录申请结果、更新追踪表，并反向校准匹配框架，越用越准。

8/add-template · 注册简历模板

注册自定义 LaTeX 简历 / 求职信模板，并自动测试编译是否正常。

9/add-portal · 扩展地区门户

为任意地区的职位门户一键生成搜索插件，让框架覆盖全球市场。

三、一次投递的 6 步流水线

最核心的 /apply 不是「生成一份简历就完事」，而是带关卡的生产级流水线：

匹配评估→ 简历定制→ 求职信生成→ 审稿人评审→ PDF 编译→ ATS 验证
  • 匹配评估：先判断这个岗位和你到底 match 到什么程度，再决定要不要投入。
  • 简历定制：针对该 JD 重写 / 调整简历，突出相关经历，而非一份通投。
  • 求职信生成：按岗位语气与要求生成定制求职信。
  • 审稿人评审：模拟审稿人视角检查一致性、夸大、遗漏，给出修改建议。
  • PDF 编译：基于 LaTeX 模板编译出排版干净的成品。
  • ATS 验证：检查 PDF 文本层能否被 ATS（ applicant tracking system）正确解析，避免「投了但机器读不出」。

「投出去前先过 ATS 文本层」——这一步是很多人海投石沉大海的隐形原因，而它把它做进了流水线里。

四、快速开始

前置依赖（门槛不算低，但都是一次性配置）：

1. Fork + Clone

gh repo fork MadsLorentzen/ai-job-search --clone  →  cd ai-job-search

2. 安装门户 CLI 依赖

for portal in jobbank-search jobdanmark-search jobindex-search jobnet-search linkedin-search; do ... ; done

3. 前置环境

Claude Code CLI（运行载体）· Python 3.10+ · Bun（门户 CLI 运行时）· LaTeX 发行版（TeX Live / MacTeX / MiKTeX，含 lualatex、xelatex）· poppler-utils（ATS 检查可选，macOS: brew install poppler）

五、适用场景

日常求职：早上开 Claude Code，/scrape 扫今天新岗 → /rank --top 10 排序 → 选最心仪的一个 /apply → 六步跑完拿到两份 PDF（CV + Cover Letter）→ 投出去后 /outcome 记录。

面试准备：/interview 基于已跟踪的申请存档，构建阶段专属准备包——STAR 示例映射（你的每条经历对应哪些行为问题）、公司与面试官调研、模拟面试角色扮演、以及诚实的差距桥接。

技能差距规划：/upskill 分析你的档案和目标职位的差距，生成带优先级的技能热力图与学习计划。

多市场求职：/add-portal 让框架覆盖任意地区——无论丹麦、中国还是德国，一条命令生成当地职位门户的搜索技能插件。

六、我的判断

适合：愿意配置终端环境、想把求职「工程化」的人；需要多市场 / 多岗位投递、且重视数据隐私（全流程本地运行）的人；受困于「每份岗位都要重写简历」重复劳动的人。

注意：① 强依赖 Claude Code + 本地环境配置，纯小白上手有门槛；② 编译 PDF 需要 LaTeX 发行版，没装就出不了成品；③ 框架本身不做自动投递，每一步都需你人工确认，定位是「减少重复劳动」而非「代投机器人」；④ 内置门户偏丹麦市场，中国区（如 Boss 直聘 / 猎聘）需自行 /add-portal 扩展。

定位：它不是海投外挂，而是把求职变成一条可审计、可迭代、数据私有的本地流水线。如果你投入的是「值得认真准备」的岗位，它会把最耗时的定制与检查环节自动化，让你把精力放在决策上。
