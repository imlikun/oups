△ AI 自动配图 — 让小黑插画帮你的文章生成手绘风格图解

传统配图 = 去家具城买现成的沙发——颜色可能不搭，但凑合能用。小黑插图 = 雇佣一位懂建筑结构的插画师——他根据你的承重墙（核心逻辑），现场画一张风格统一的装饰画。

一、这是什么？

想象你刚写完一篇 3000 字的技术文章，结构清晰、逻辑自洽，但总觉得少了点什么。你试过放一张 Unsplash 的氛围图，但那张图跟内容毫无关系；你也试过画流程图，但读起来像产品说明书。

它不是通用插画工具，也不是 PPT 信息图模板。它的核心目标只有一个：先理解文章里的认知锚点，再把其中一个关键动作画出来。

三、核心能力详解

3.1 「小黑」IP：荒诞的系统参与者

小黑不是吉祥物，不是贴纸，不是站在角落里的装饰物——它是一个正在认真参与系统运转的荒诞工作者。

  • 外形：黑色实心身体、白点眼睛、细腿、空表情
  • 角色定位：不是卖萌，而是「认真干活的荒诞小角色」
  • 核心规则：如果去掉小黑，画面仍然完全成立，说明小黑太装饰了，不符合要求

3.2 视觉风格规范

规范项具体要求
背景纯白，无纸纹、无米色、无阴影、无渐变
线条黑色手绘线稿，细线，轻微抖动
留白主体只占画面 40%-60%，大量留白
批注仅少量红色、橙色、蓝色中文手写批注
比例16:9 横版，专为正文阅读体验优化
单图单锚点一张图只表达一个核心动作、结构、状态或隐喻

3.3 认知配图哲学

市面上大多数 AI 配图工具的逻辑是：你给 prompt，我出图。小黑的逻辑是：先理解文章 → 提取认知锚点 → 发明物理隐喻 → 再画图。

这个「认知提取」环节确保的是「图在解释文章」，而不是「图在装饰文章」。

四、9 步工作流程详解

  1. 读取文章

    Markdown / Notion / 截图 / 用户给的主题

  2. 提炼核心

    找出文章里的认知转折、关键判断、流程结构

  3. 输出 shot list

    每张图只对应一个认知锚点（4-8 张）

  4. 选择结构类型

    从 8 种类型中匹配（参见第七节）

  5. 发明物理隐喻

    低科技、怪诞但成立的场景——如「堆积木」表示「信任构建」

  6. 小黑承担核心动作

    小黑必须在画面里「干活」，不是装饰

  7. 逐张生成图像

    调用图像模型，每张图单独生成

  8. QA 检查清单

    6 项验证，不通过就重生成

  9. 保存并报告

    PNG 保存到 assets/<article-slug>-illustrations/

QA 检查清单（6 项）

  • 纯白背景？
  • 留白足够？
  • 小黑在承担核心动作（非装饰）？
  • 中文标注存在且合理？
  • 没有 PPT 信息图的感觉？
  • 没有复刻示例图的构图？

五、3 种使用方式

方式一

只做配图规划（推荐先用这个）

适合：不确定文章哪里该配图，想先让 AI 分析一遍

方式二

直接生成正文配图

适合：文章已经定稿，直接出全套配图
Use $ian-xiaohei-illustrations 把下面这篇文章生成 4 张小黑怪诞正文配图...
方式三

为单个概念/金句生成一张图

适合：社交媒体分享、演讲稿配图、单观点视觉冲击力
Use $ian-xiaohei-illustrations 为「信任不是喊出来的，而是一块证据一块证据铺过去」生成一张正文配图...

八、注意事项与避坑指南

坑原因解决方案
中文文字错字AI 图像模型对中文渲染不稳定中文越短越稳定；错字严重时减少标注词并重生成
风格漂移模型不严格遵循风格约束Prompt 中反复强调「纯白背景、黑色手绘线稿、仅红橙蓝三色」
小黑形象不一致随机种子导致固定 Seed 值，强调「黑色实心、白点眼、细腿」三要素
色彩失控模型加入绿色、紫色等杂色增加负面约束：「禁止使用绿色、紫色等其他颜色」
小黑太装饰去掉小黑画面仍然成立检查小黑是否承担了核心动作
一张图塞太多违反单图单锚点原则拆分成多张图，每张只讲一个核心结构

核心原则：图片里的中文文字越短越稳定。每张图只讲一个核心结构。小黑必须承担核心动作——如果去掉小黑画面仍然完全成立，说明小黑太装饰了。

十二、常见问题 FAQ

Q1：这个 Skill 适合什么人？

适合写中文观点类、方法论类、经验复盘类文章的人。如果你的内容有大量抽象判断和认知转折，小黑能帮你把关键观点钉在读者脑子里。

Q2：跟 Midjourney / Stable Diffusion 有什么区别？

本质区别是出发点不同。通用工具是「你描述画面，我画出来」。小黑是先理解你的文章，再为关键认知锚点发明视觉隐喻。前者是画具，后者是认知配图师。

Q3：不用 Codex 能用吗？

目前不行。小黑插图是基于 Codex 的 Skills 机制设计的。如果你不用 Codex，可以手动复制 README 中的核心指令作为系统预设，但效果可能不如原生 Skill 稳定。

Q4：生成的图可以商用吗？

项目采用 MIT License，可以自由使用。但注意底层图像模型的输出需遵守该模型的使用条款。

Q5：中文错字严重怎么办？

优先减少标注词并重生成。如果仍不稳定，建议只生成图像主体，文字批注后期通过代码或 PS 手动添加。

Q6：支持英文文章吗？

专为中文文章优化。若用于英文，需调整 Prompt 中的文化隐喻部分，否则可能出现理解偏差。

十四、一句话总结

小黑插图不是「给你一支万能画笔」，而是「给你一个风格固定的画室」——在这个画室里，AI 先读懂你的文章，再为每一个关键认知动作，画一张有记忆点的怪诞手绘解释图。

如果你在做长期的中文内容输出，需要一套稳定的、有识别度的、能解释文章而非装饰文章的配图体系，小黑插图可能是你内容生产线上缺的那一环。
