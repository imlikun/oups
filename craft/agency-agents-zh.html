【WB】266 个 AI 专家任你调遣：agency-agents-zh 把 The Agency 汉化 + 本土化

agency-agents-zh 是 The Agency 的中文增强版，把 AI 专家角色从 144 扩展到 266 个，新增 50 个中国市场专属智能体，一句话即可调度多专家协作。

一、这是什么

先给一句话结论：agency-agents-zh 不是一份「通用提示词合集」，而是一套「开箱即用的 AI 智能体角色库」——每个角色都有独立人设、专业流程和明确交付物。

它的上游是 msitarzewski/agency-agents（GitHub 约 127k Stars 的爆款项目，原名 The Agency）。中文版由社区维护者 jnMetaCode 在完整翻译上游的基础上做了本土化增强，所以名字里的「zh」就是 Chinese 的意思。

普通提示词只告诉 AI「你是一个专家」；而这里的智能体定义了专家怎么思考、怎么做事、交付什么。例如安全工程师会按 OWASP Top 10 逐项审查代码，小红书运营专家会直接输出完整的种草笔记策略 + 达人合作方案。装进你的 AI 编程工具后，用自然语言点名就能激活。

本文交付物

一篇帮你判断是否值得装、以及「装完先玩哪几个」的测评笔记。坤哥说后面自己玩一玩，所以第四章我专门替你梳理了上手路线，照着复制就能跑。

二、核心能力拆解

别被「266 个」这个数字吓到，它其实是由两部分拼起来的，理解结构比记数字更重要。

1215 个 = 上游翻译

把英文原版 The Agency 的 215 个智能体完整汉化，保留原人设与流程，中文场景下读起来不再拗口。

250 个 = 中国市场原创

这是中文版真正的增量价值：小红书 / 抖音 / 微信 / B站 / 飞书 / 钉钉等平台运营，以及跨境电商、政务 ToG、医疗合规、Qt 工业上位机、机械设计、畜禽养殖档案核对等垂直领域。

覆盖 20 个部门（工程、设计、营销、产品、游戏、安全、GIS、金融、HR、学术等），并支持 18 种 AI 工具接入——Claude Code、Cursor、Copilot、Codex、Hermes Agent、OpenClaw 等主流编程助手都能直接挂载。

最关键的放大器是配套编排器 agency-orchestrator（简称 AO）：你一句话下任务，它能让多位专家按 DAG（有向无环图）自动分工协作，几分钟内交付一份「多专家合力」的完整方案，而不是单打独斗。

三、怎么装到你的工具

它有四种上手方式，按「想多快用上」从易到难排：

方式一一键脚本（推荐先试）

仓库自带 install.sh，自动检测你本机装了哪些 AI 工具，一键注入全部角色。

# 克隆仓库
git clone https://github.com/jnMetaCode/agency-agents-zh
cd agency-agents-zh
# 自动检测已装工具并安装（如 Claude Code / Cursor / Copilot）
./scripts/install.sh
# 或指定装到某个工具，例如 OpenClaw / claude-code / cursor
./scripts/install.sh --tool claude-code
方式二桌面客户端（免装 Node）

原生 App，macOS / Windows / Linux 都能跑，不用先配 Node 环境。适合「只想马上点开玩」的人。在线体验入口：ao.aiolaola.com/experts。

方式三在线体验（零安装）

不想动本地环境，直接到 agents.vibed-lab.com 搜 jnmetacode-agency-agents-zh 即可在网页里试跑角色，适合先观望再决定要不要装。

方式四手动复制

把仓库里的 *.md 拷到 ~/.claude/agents/（或项目的 .claude/agents/），新开一个会话就能加载，适合想精细控制装哪些的人。

坑：装完识别不到

刚复制进去时，当前会话可能读不到新角色。开一个新的 Claude Code / Cursor 会话（或重启），让它重新加载 agents 目录即可。这是最常见的新手卡点。

四、玩一玩：八戒替你梳理的上手路线

坤哥说后面自己玩，那我先把「先玩哪几个、怎么玩出东西」给你排好。下面是基于官方 README 与角色定义梳理的可复制路线，我没有在你的机器上真跑（环境不在我这边），但每一步都是官方工作流，你复制就能用。

1）先玩 3 个「立刻能见产出」的角色

  • 小红书运营专家：丢一个产品 / 主题进去，它会产出完整的种草笔记策略 + 标题库 + 达人合作建议。对坤哥做内容分发最直接。
  • 安全工程师：把一段代码或仓库路径给它，按 OWASP Top 10 逐项审，输出风险清单 + 修复建议。适合评估自己项目。
  • 跨境电商文案 / 选品：给品类，产出多平台 listings、卖点提炼、合规提示。中文版 50 个原创里最实用的几块之一。

2）再玩编排器：一句话调度多专家

单个角色只是「请了个专家」，编排器 AO 才是「开了家咨询公司」。试试这种句式：

# 用 AO 编排多专家协作（示例任务）
用安全工程师 + 小红书运营 + 跨境电商文案 三个角色，
帮我为「一款国产机械键盘」出一版：安全合规自查清单 +
小红书种草笔记策略 + 跨境电商上架文案，按 DAG 串行交付。

预期产出是一份「三专家接力」的方案：安全先排雷 → 运营定调性 → 文案出 final 稿。几分钟级别，比你自己一个个问快得多。

3）进阶：把角色库当「方法论外挂」

项目 README 里提到配套 180 节免费 AI 编程课 +《AI 编程实战三卷书》。把 266 个角色装进 Claude Code / Cursor 后，再配合这套方法论，效率比单用角色高一个档次——角色解决「谁来做」，方法论解决「怎么做得好」。

玩的顺序建议：先单角色出成品 → 再 AO 编排协作 → 最后配方法论沉淀。别一上来就装 266 个全量，按需取用才不会看得头晕。

五、坑与提醒

测评里该泼的冷水也得泼，免得你踩完回头骂我。

注意点说明
上游更新更快中文版基于某一版上游翻译 + 本土增强，上游若有大更新，中文版同步会有延迟，关注仓库 commit 即可。
客户端 vs 源码桌面客户端免装 Node、上手最快，但深度定制（改人设 / 加字段）还是得用源码仓库。
Star 数口径上游 127k、中文版约 16.6k（2026-07-05），引用时别把两者混为一谈。
密钥安全角色接大模型 API 时涉及 Token / Key，注意别把凭证写进会提交的 md 文件里；本地用环境变量更稳。
别全量装266 个全装进去，会话加载和检索都会变慢，按需挑部门装体验更好。

六、获取与结语

一句话总结：如果你想要「开箱即用的专家团队」而不是从零写提示词，agency-agents-zh 是目前中文圈最完整的那一个——尤其是 50 个本土原创角色，是英文原版给不了的东西。
