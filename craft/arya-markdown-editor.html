一、它是什么

先说清楚边界：Arya 是一个「在线 Markdown 编辑器」，不是笔记软件、也不是静态博客生成器。它解决的是最朴素的需求——打开浏览器就能写 Markdown，写完能预览、能导出，不用装任何客户端。

它由独立开发者 nicejade 创建，GitHub 3.8k Stars，MIT 协议，版本 v3.8.0.0。项目地址 nicejade/markdown-online-editor，在线体验站 niceshare.site。它的卖点很直白：浏览器打开即用。

一句话定位：一个纯前端的在线 Markdown 编辑 / 转换 / 导出工具。写完即预览，支持流程图、甘特图、五线谱等扩展语法，还能把 HTML 反向转成 Markdown，最终一键导出多种格式。

二、核心编辑能力

Arya 不只是「纯文本 + 预览」，它基于 Vditor 渲染引擎，支持一批扩展语法，覆盖技术写作与知识整理的常见需求：

  • Markdown 基础：标题、列表、表格、代码块、数学公式等标准语法齐备；
  • 流程图（Flowchart）：用文本描述直接渲染流程图，适合画架构与流程；
  • 甘特图（Gantt）：项目排期、任务依赖一图说清；
  • 五线谱（Music / ABC 记谱）：支持乐谱渲染，对音乐类笔记友好；
  • HTML 自动转 Markdown：把网页复制来的富文本 / HTML 粘贴进去，自动规整成 Markdown——这是它相对很多轻量编辑器的实用差异点。

对「从网页抄了一堆格式乱掉的文本、想快速变成干净 Markdown」的人，HTML→Markdown 这一步能省掉大量手动清理时间。

三、多格式导出

写完之后，Arya 支持把当前文档导出为多种格式，覆盖大多数交付场景：

PDF· PNG· JPEG· Word（.docx）
  • PDF / PNG / JPEG：适合把文档 / 长图直接分享出去，或贴进汇报；
  • Word：对接需要走 Office 流转的协作场景，避免再开工具转格式；
  • 支持导入的格式也较宽：Markdown、INI、XML、TXT、PNG、JPEG、GIF、HTML 等。

四、技术栈与体验

1Vue2 + Vditor 构建

前端用 Vue2 框架、Vditor 作为 Markdown 渲染与编辑内核，纯前端运行、不依赖后端服务。

2内容本地存储

编辑内容存于浏览器本地，关页不丢；适合随写随存、不强制登录的轻量写作。

3实时预览 + 快捷键

左写右览的实时预览，配合常用快捷键，写作流不被「点按钮」打断。

六、我的判断

适合：想要「开箱即写、写完即导出」的轻量写作者；常从网页搬内容、需要 HTML→Markdown 清理的人；需要偶尔出 PDF / 长图 / Word 但不想装重型软件的用户。MIT 协议，可自由修改与自托管。

注意：① 技术栈仍是 Vue2，相对当下主流 Vue3 偏老，长期维护节奏需自行评估；② 它是「编辑器」而非「笔记系统」，没有跨设备同步与知识管理，重协作请另选；③ 内容默认存浏览器本地，换设备 / 清缓存需先导出备份；④ 3.8k Stars 属稳健小工具量级，不是大型社区项目。

定位：它把「写 Markdown」这件事压到了最低门槛——一个网址、打开就写、写完能出多种格式。对「不想为写作再装一套环境」的人，是称手的临时工作台；要深度知识管理，则另有其选。
