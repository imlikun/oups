一、这是什么？

Chatwoot 是一个开源的全渠道客户支持平台（Omnichannel Helpdesk）。

简单来说：你开公司做电商/SaaS/在线服务，需要一个统一的客服工作台收邮件、收网站聊天、收 WhatsApp 消息、分配给客服同事处理——Chatwoot 就是干这个的，而且免费、可自托管、数据完全自己掌控。

二、它解决什么痛点？

商业客服系统有几个老问题：

  1. 贵——Intercom、Zendesk 按坐席数收费，10 个客服一年轻松过 10 万
  2. 数据不在自己手里——客户对话数据存在第三方服务器
  3. 不能深度定制——业务流程跑通了想改，告诉你这是「企业版功能」
  4. 渠道割裂——邮件、聊天、WhatsApp 在不同系统里切换

Chatwoot 一次解决：

  • 零授权费——MIT 协议，源码可改可商用
  • 自托管——Docker 部署在自己的服务器
  • 全渠道统一——一个工作台收所有渠道消息
  • 可深度定制——源码开放，业务流程自己改

五、怎么部署？

方式一：Docker（推荐）

git clone https://github.com/chatwoot/chatwoot.git
cd chatwoot
cp .env.example .env
# 编辑 .env，配置数据库密码、Redis 等
docker-compose up -d

方式二：自托管 systemd

仓库 deployment/ 目录提供完整 systemd service 配置（web + worker），适合生产环境裸机部署。

cwctl upgrade  # 一键升级、备份、监控

方式三：Heroku / Clever Cloud

仓库根目录有 app.json，点击 Heroku Deploy Button 就能部署。

方式四：GitHub Codespaces

仓库有 .devcontainer/ 配置，可直接在浏览器里试用开发环境。

方式五：官方 Cloud 版

chatwoot.com 提供托管服务，按月付费，不想运维的选择。

六、谁在用？

  • 初创 SaaS 公司——客服数据是核心资产，自托管避免被绑死
  • 跨境电商——WhatsApp/Facebook Messenger 是海外客户主渠道
  • 隐私敏感行业——医疗、金融、法律，必须自托管
  • 开源项目作者——免费、可自托管，社区项目首选
  • 出海团队——多语言 + 多渠道，一个工具覆盖全球客户
  • 不想被 Zendesk 绑架的团队——商业客服系统涨价频繁

    • 不太适合

    • 完全没技术团队的公司——建议用 Cloud 版
    • 10 人以下超小团队——免费版 SaaS 可能更省事
    • 只要邮件工单——简单邮件转发规则就够

    九、避坑指南

    自托管前必看

    • 资源需求——至少 2 核 4GB 内存（推荐 4 核 8GB），PostgreSQL + Redis + Elasticsearch 三件套都要装
    • Elasticsearch——至少 1GB 堆内存，文档量大需要更多
    • CDN 必备——网站聊天 widget 加载慢会影响转化
    • 备份策略——PostgreSQL 每日备份，Elasticsearch 定期重建

    升级注意

    • 项目走 develop 分支持续滚动，生产环境建议用 tags 而非分支
    • cwctl 工具是自托管升级利器，能自动停服、迁移、启动
    • 升级前看 CHANGELOG，注意数据库迁移

    商业使用

    • MIT 协议可商用——但要在 About 页面保留版权声明
    • AI Agent (Captain) 接入 OpenAI 需要海外信用卡，国内团队可换成 DeepSeek/通义千问
    • 中文支持需要单独配置翻译文件（默认 i18n 不全）
    • 移动端只有响应式 Web，没有原生 App

    十二、总结

    Chatwoot 的核心价值：

    • 不是更便宜，而是更可控——数据在自己手里，业务流程自己定
    • 不是更多功能，而是更开放——MIT 协议 + 完整 API = 无锁定
    • 不是社区玩具，而是企业级产品——31K Stars + 几乎每天更新 + 完整企业版
    如果你的客服数据是核心资产，Zendesk/Intercom 的授权费+数据托管是隐形税，Chatwoot 是开源世界里最成熟的替代方案。
