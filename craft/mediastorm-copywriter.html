mediastorm-copywriter：用影视飓风风格批量产出中文口播稿

一、它是什么

先说清楚边界：它不是视频生成器，而是一个「口播稿生成 Skill」——给 Claude Code 用的一种技能插件。你给它一个选题，它产出一篇「影视飓风味」的中文口播稿，画面、剪辑、配音都不归它管。

它的目标很具体：把影视飓风视频里「为什么好看、为什么抓人」的规律，从「只可意会」变成「可显式复用」的写作配方。你不需要反复研究人家怎么写，直接让 AI 按配方出第一稿。

一句话定位：把影视飓风 19 条视频口播稿的「结构规律」提炼成 Skill，让 AI 按影视飓风味产出成稿。仅生成稿子，不碰画面与剪辑。

二、背后的方法：19 条视频的结构化分析

这个 Skill 不是凭感觉写的提示词，而是建立在对影视飓风最新 19 条视频口播字幕的结构化分析之上。它从真实样本里提炼出几个可复用的维度：

  • 开头钩子：用什么方式在 3 秒内把人留住；
  • 黄金七段式骨架：一篇稿子的主干结构（见下节）；
  • 节奏 / 转场：段落之间怎么过渡才不塌；
  • 数据换算 / 类比：抽象数字怎么变成「人话」；
  • 金句与 CTA：收尾怎么升华、怎么引导互动。

规律提炼自影视飓风公开视频的自动字幕。Skill 内仅含用于说明规律的「短句引用」，完整视频与口播内容版权归影视飓风所有——它定位是「学习规律、二次创作参考」，不是照搬原稿。

三、一篇稿子的骨架：黄金七段式

核心骨架叫「黄金七段式」，一篇合格口播稿按这个顺序铺开（评测 / vlog 类有变体）：

钩子→ 升维→ 植入→ 分点主体→ 转折→ 升华→ CTA
  • 钩子：开头 3 秒抓住注意力（见下节 6 类钩子）；
  • 升维：把具体话题抬到更大的视角，制造「值得看下去」的悬念；
  • 植入：自然地带出合作方 / 产品，不硬切；
  • 分点主体：核心信息分点展开，配数据换算与类比；
  • 转折：用反差 / 意外推进节奏，避免平铺直叙；
  • 升华：金句收尾，把话题拉回价值观或情绪高点；
  • CTA：标志性收尾（如「点点小心心…那么我们下次再见」）。

四、关键能力

16 类开头钩子

实物特写 / 反常识设问 / 观众代入 / 极端场景 / 人物前置 / 栏目仪式——按需选用，避免每篇都用同一种开场。

2强第二人称「你」+ 自问自答

通篇用「你」拉近距离，并以自问自答制造对话感与节奏，避免念稿腔。

3数据必换算 · 概念必打比方

抽象数字换算成可感知的量（时间、金钱、体积），抽象概念用生活类比落地，降低理解门槛。

4金句收尾 + 标志性 CTA

结尾给一句能截图的金句，并带上辨识度高的行动召唤，强化账号记忆点。

五、安装与使用

它是一个标准 Claude Code Skill，安装即用：

1. 克隆到 skills 目录

git clone git@github.com:imfangli/mediastorm-copywriter.git ~/.claude/skills/mediastorm-copywriter

2. 触发

对 Claude 说：「用影视飓风风格写一篇关于 XX 的口播稿」（可选指定内容类型 / 时长 / 合作方 / 素材）

3. 结构

SKILL.md（入口 + 10 条硬规则 + 自检清单）＋ references/（structure.md 骨架、hooks.md 钩子、transitions.md 转场、style-guide.md 语气、examples.md 真实片段拆解）

六、我的判断

适合：做科技 / 数码 / 测评 / 知识类口播、想稳定产出「影视飓风味」的人；已经把 Claude Code 当工作流载体、希望写作也「技能化」的人。

注意：① 仅生成口播稿，不生成视频 / 画面，下游还需自己拍剪；② 规律提炼自公开字幕，二次创作需规避直接照搬，尊重原 UP 主版权；③ 单 commit 早期项目（2026-07-14），MIT 协议，迭代空间大；④ 本质是 Prompt / Skill 工程，依赖 Claude Code 运行。

定位：它把「好口播的结构」显式化、可复用化——不是替你创作，而是给一个稳定高质量骨架，让你把精力放在选题与信息本身。对「知道影视飓风好看、但写不出那味」的人，是低门槛的起步杠杆。
