## 口播文案写作规范

1. **开头 3 秒必须抓人**
   - ✅ "你以为 xx 其实 yy" / "90% 的人都不知道" / "省下 xxx 的方法"
   - ❌ "今天我们来聊聊 xx"（太平淡）

2. **每句话不超过 25 个字**
   - 视频号是碎片化消费，长句 = 划走

3. **说人话，别念论文**
   - ❌ "该模型采用 transformer 架构的注意力机制实现"
   - ✅ "简单说就是，它读一段话就能抓住重点"

4. **数字具象化**
   - ❌ "大幅提升效率"
   - ✅ "原来要 3 小时的活，现在 10 分钟搞定"

5. **段尾留钩子**
   - 每个要点结尾暗示"还有更狠的"，拉住完播

## 画面制作建议

| 要素 | 说明 |
|------|------|
| **分辨率** | 1080×1920（9:16 竖屏） |
| **字幕** | 必须加！60% 用户静音观看；字号 ≥ 36px，底部居中，带半透明背景条 |
| **节奏** | 每 3–5 秒切一次画面，静止超过 8 秒 = 掉关注 |
| **配色** | 与文章模板色系一致（蓝 A / 紫 B / 青绿 C） |
| **封面上字** | ≤ 8 个大字，字号占画面 1/3，对比色白字/黑底或黄字/深底 |
| **真人出镜** | 可选。不出镜则用「封面图 + 数据动效 + 关键词大字」替代 |

## 从文章提取脚本的快捷流程

```
1. 读文章 → 提炼 1 句钩子（最反直觉/最有价值的那句）
2. 找 3–6 个核心要点（按重要性排序）
3. 给每个要点配：
   - 口播版（口语化，≤ 25 字）
   - 画面建议（截图？大字？表格？）
4. 写 CTA（与文章栏目关联："关注我，xx 栏目每天更新"）
5. 总时长控制：正常语速 4 字/秒，据此删减
```
